0x01 little_tommy

题目分析

拿到elf文件，查看信息如下，为32位动态链接程序，开启NX保护。

直接拖到IDA里分析，程序为典型的功能式菜单。

下面对各具体功能进行分析：

Create account

该功能为：从控制台读取输入，分别作为First Name和Last Name存储在堆中，此堆地址存储在0x0804A048 main_account处，堆状态如下：

Display account ：将存储的信息打印出来，此功能不加赘述

Delete account ：

删除当前信息，该功能存在一处漏洞，即在free掉内存堆块后，堆块指针没有置零。

Add memo ：

输入一串字符串，然后进行字符串拷贝操作。此功能的关键在于strdup函数：

strdup为字符串拷贝函数，其内部调用malloc为变量分配内存，将字符串拷贝到新建的位置处。该函数一般和free函数成对使用。

Print flag：

当if语句的条件成立时，则会成功拿到flag。

解题

基于上述分析可知，破解此题的关键在于满足print flag功能中的if语句中的两个条件：

由于Delete account在执行完free操作后，并没有把堆块指针置零，因此main_account中将一直存储有一个堆块指针，即条件1在执行完Create account操作后将会一直成立。

条件2即要求在申请的堆块的 0x40 偏移处的内存满足字符串 kcuf，即：

由此自然想到在Create account中通过控制输入，在main_account+0x40处写入指定字符串。
但是，在上述对Create account的分析发现，Create account内部通过对字符串拷贝的限制和截止符的运用，使得其操作无法达到main_account+0x40内存处，因此要另辟蹊径了。

我们把目光转向Add memo，在这个函数内部，包含字符串输入和字符串拷贝操作，且字符串拷贝函数strdup内部调用了malloc。因此可以提出这样的思路：

  1. 调用Create account，申请堆块，输入信息
  2. 释放掉上述堆块（由于上述堆块紧挨着top chunk，因此在free掉之后，会合并到top chunk）
  3. 调用Add memo，控制输入的字符串内容，利用strdup内部的malloc操作，将步骤2中释放的内存区域从top chunk上切割下来，利用strdup的拷贝操作直接在指定区域填入指定的字符串kcuf。
  4. 调用print flag，if条件成立，拿到flag。

0x02 ropme

题目分析

为64位程序，有NX保护。

从反编译代码看，s位于rbp-0x40位置处，而fgets允许输入500个字符到s中，程序存在明显的栈溢出漏洞。

可通过覆盖返回地址控制eip劫持程序流程，但是由于NX保护，不能在栈中直接打入shellcode执行，且程序中自身没有system函数的调用，因此本题目的整体思路是采用ret2libc的方式。

解题

ret2libc 即控制程序去执行libc中的函数，通常情况下选择system("/bin/sh")。

整体的思路是：

  1. 利用程序漏洞泄露某个libc函数的地址
  2. 泄露libc的版本信息，获取system和/bin/sh的地址
  3. 再次执行程序触发栈溢出，执行system("/bin/sh")

由于本程序为64位程序，不同于32位的栈中存参数的方式，根据其调用约定，前六个参数存储在 RDI, RSI, RDX, RCX, R8 和 R9 寄存器中。收集一下程序中可用的gadget，这里具有使用价值的是0x4006d3 pop rdi; ret，我们可以在栈中依次放入argc_addr和api_addr，则这段gadget则相当于执行api(argc)。
