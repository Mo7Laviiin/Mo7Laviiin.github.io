拿到user.txt

下一步尝试提权操作

首先上传脚本，搜集靶机内相关信息：

发现靶机内有一ovrflw程序，具有SUID标志位

文件为32位程序，运行程序，提示输入字符串

这里猜测，此程序可能存在overflow溢出漏洞

把程序拿回本地进行调试分析

查看程序保护措施

main函数反汇编代码如下：

在0x080484c0观察到strcpy函数的调用，猜测潜在栈溢出

输入字符串，尝试计算偏移量：

得到偏移量为112字节

由于开启NX保护，因此不能直接将shellcode打到栈上

因此，尝试利用libc内的system和“/bin/sh”，即ret2libc

首先搜集地址信息：

构造payload如下：

'a'*112 + 0xb756a310 + 'bbbb' + 0xb768cbac

靶机开启了地址随机化ASLR的保护机制：

因此libc的基址会不断变化，因此需要利用上述payload进行爆破

当libc加载的基址刚好满足payload的条件时，成功溢出并执行system('/bin/sh')

循环多次后，成功拿到shell
